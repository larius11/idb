About our website

On The Run is a website that allows people to draw interesting connections between the FBI Most Wanted Page and information on different types of crimes as well as information on the states in which these crimes were committed.

  • Intended Users: Anyone interested in learning about crime/criminals
  • Explanation: This integration of disparate data allows people to not only see specfic information about each criminal, but also a macro look at crime in the states in which they committed their crimes.
  • Group Name: The Slackers
  • Group Members: Ricardo Riveron, Roshan Dongre, Zara Louis, Ramon Perez, Krishna Ramdeep

  • Data Sources Used And How They Were Scraped:
    • FBI Crime Stats API
        • We used the requests module to scrape this data using python. This API provided useful crime statistics shown on each of our model pages.
    • FBI Most Wanted API
        • We used the requests module to scrape this data using python. This API provided information on the criminals on our webpage.
    • Google Maps API
        • We got an API key from Google Maps and then embedded the code provided in our HTML. This provided us with maps on our model pages.
    • We also are going to be using the Google News API, World Bank API, and potentially other APIs as we further develop the project
  • Tools Used: Flask, Requests, Bootstrap, Travis CI, Postman, Slack, GitHub
    • Flask- The core of our project as it handles the API data requests and acts as an interface that communicates with the front-end of our application
    • Requests- The requests module is what we are using to handle parsing the information from the APIs we are using
    • Bootstrap- The Bootstrap template is the core of our front-end for this project and is the main framework used by most front-end developers
    • Travis CI- While we did not use Travis CI directly for this phase of the project, we did set it up as it will be a useful tool during the next phases as we build out our website and integrate testing. Travis CI is a great continuous integration tool that is very useful in robust applications such as ours
    • Postman- Postman is a tool that allows one to build and test APIs and we used it for creating our API for this project. While we did not implement testing for this phase we will be using it extensively in the upcoming phases, notably IDB2
    • Slack- Slack is a messaging service that allows software developers to share and discuss code and ideas, with the great feature of GitHub integration. We used Slack extensively in this phase and it was our primary means of communication
    • GitHub- GitHub is a open source code sharing platform that we used to store our code and coordinate with each other
